# Brushforge Documentation

This folder contains project documentation, analysis reports, and reference materials.

## Contents

### Project Overview
- **CONTEXT.md** - Comprehensive project context and architecture overview
- **PERFORMANCE_GUIDE.md** - Performance optimization guidelines and best practices

### Feature Documentation
- **RECIPES_FEATURE.md** - Documentation for the recipes/palettes feature
- **CONVERTER_IMPLEMENTATION_GUIDE.md** - Guide for the paint converter feature
- **CONVERTER_FINDINGS_SUMMARY.md** - Analysis of converter implementation

### Code Reviews
- **PALETTES_CODE_REVIEW.md** - Code review notes for palettes feature

### Technical References
- **ANALYSIS_INDEX.txt** - Index of analysis documents
- **README_CONVERTER_ANALYSIS.md** - Converter feature analysis README
- **data_flow_diagram.txt** - Data flow architecture diagram

### Build Logs
- **build.log** - Recent build output (for debugging)

## Usage

For developers working on Brushforge:
1. Start with **CONTEXT.md** for project overview
2. Refer to **PERFORMANCE_GUIDE.md** before optimizing
3. Check feature-specific docs when working on that area
4. Update docs when making significant changes

## Maintenance

- Keep CONTEXT.md updated with major architectural changes
- Document new features with their own markdown files
- Archive old analysis documents when no longer relevant
